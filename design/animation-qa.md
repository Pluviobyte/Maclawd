# Maclawd animation QA — V5.3 — 2026-07-17

本轮根据评审反馈，把质量基线从“每个动作都有完整小场景”修正为“身体先演，道具只做必要提示”。23 个动作全部取消完整横向底座，**Top-down Sleep** 也删除了床垫，只保留贴身枕头、毛毯和 Zzz。

V5.3 不再只压缩宽度，而是删除构成横线的物品：地毯、低桌、路径、餐垫、软垫、地面条、晾衣绳、横向木杆和睡眠床垫。96px 渲染下，23 个当前动作不再含任何完整底座。

## 自动检查

- 26 个 SVG 源文件全部通过 XML 解析（23 个当前动作、1 个历史兼容入口、2 个机械原型）。
- 23/23 当前动作保留 `viewBox="-15 -25 45 45"`。
- 23/23 当前动作保留身体、双爪、四腿、双眼的固定源矩形。
- 23/23 当前动作使用身体色 `#DE886D` 与眼睛色 `#000000`。
- 全套 V5.3 在 96px 共渲染 979 个确定性 Chromium 帧，并以相同时序渲染 64px 检查帧，输出 23 个 GIF。
- 23/23 GIF 至少包含两个不同实际帧；没有静态假 GIF。
- 23/23 动作满足 `max-width <= 48px` 的新横向包围盒硬门。
- 21 个循环/反应 GIF 首末采样帧像素闭合；`waking` 是非循环转场，`drop` 的可见 alpha 首末一致，仅透明像素 RGB 有编码差异。
- 三份机器可读 JSON 合同通过语法解析。
- 动态实验室加载 23 个当前动作；`working.command` 映射到 generic `working`，不重复占一个视觉动作。

## 视觉检查

- 在 64px 与 96px 两档检查身体轮廓、双眼、场景识别、接触关系、横向包围盒与裁边。
- `idle`、`success`、点击、双击、落地和低电量为纯身体动作，不再依赖软垫。
- 拼图、工作牌、书、信纸、积木、测试玩具和同步线轴都与身体重叠、贴在单侧或纵向叠放。
- `delegating` 删除地面路径，助手纵向接力；`needs_owner` 删除餐垫；`error` 删除地面条，仅保留会扣住身体的篮子。
- `drag` 的双爪抓住中央吊环，不再抓横向木杆；`edge` 的纵向帘幕保留，因为它不造成横向铺陈。
- 未使用 UI 面板、进度条、光晕、速度线、烟雾或阴影。
- `sleeping` 按用户明确要求保留硬边像素 `Zzz`，仍是漂浮标点规则的唯一例外。
- 长页面浏览器实验室通过 23 卡加载检查；静态审查截图使用同一批确定性 96px 帧，避免离屏 `<object>` 在截图工具中不绘制。
