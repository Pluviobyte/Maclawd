# Maclawd 主状态动作圣经 v2

状态：12 个主状态已完成 SVG/CSS 动作实现与 96px 视觉审查。
完整的紧凑版静态外形评审集见
[`design/main-state-concepts.md`](main-state-concepts.md)。`thinking`、`working`、
`delegating`与 `needs_owner` 已进入趣味化 v3 动画实现；旧机械原型仅作历史对照。

## 1. 角色不可变合同

所有状态都必须复用同一套身体与眼睛几何：

| 部位 | 固定值 |
| --- | --- |
| 身体 | `x=2 y=6 width=11 height=7` |
| 左/右爪 | `x=0/13 y=9 width=2 height=2` |
| 四条腿 | `x=3/5/9/11 y=13 width=1 height=2` |
| 双眼 | `x=4/10 y=8 width=1 height=2` |
| 身体颜色 | `#DE886D` |
| 眼睛颜色 | `#000000` |

允许对身体部件所在的 `<g>` 做离散位移、旋转和缩放；不允许改写上述矩形本身，不增加描边、渐变、软阴影或替代眼睛。

## 2. 统一视觉语言

Maclawd 的当前视觉语言是「小宠物认真演一出日常小剧场」：

- 结构深灰：`#455A64`
- 深色轴心：`#1E1E2E`
- 驱动青：`#40C4FF`
- 冷光青：`#D7F3FF`
- 提示琥珀：`#FFC107`
- 浅琥珀：`#FFE082`
- 成功绿：`#66BB6A`
- 故障红：`#EF5350`

状态首先靠姿势、视线、用力和反应读出，道具只负责补充隐喻。道具应是临时出场的日常物、手工物或玩具，而不是永久安装在身上的功能面板。每个循环都必须有「准备—尝试—反应—复位」，不使用漂浮标点、文字、速度线或光晕。

## 3. 动作总表

| 状态 | 动作名 | 模式 | 时长 | 核心读法 |
| --- | --- | --- | ---: | --- |
| `idle` | Calm Calibration / 平静校准 | 循环 | 5.6s | 无配件，呼吸、观察、一次眨眼 |
| `away` | Blanket Drag / 拖毛毯 | 单次 | 3.6s | 伸展、犯困、把毛毯拖到身边 |
| `sleeping` | Blanket Burrito / 毛毯卷 | 循环 | 5.6s | 裹进同一条毛毯，闭眼慢呼吸 |
| `waking` | Blanket Pop / 毛毯弹起 | 单次 | 2.2s | 从毛毯中弹起，把两侧掀开并站稳 |
| `thinking` | Shell Shuffle / 贝壳挑选 | 循环 | 5.6s | 逐个比较三枚贝壳，最后举起中选者 |
| `working` | Token Knitting / 编织彩带 | 循环 | 4.8s | 双爪交替编织，彩带增长后得意检查 |
| `delegating` | Hatchling Parade / 小蟹出发 | 循环 | 4.8s | 三只小助手依次醒来并错相出发 |
| `needs_owner` | Stuck Jar / 拧不开的罐子 | 循环 | 4.2s | 用力尝试两次，最后把罐子递给主人 |
| `compacting` | Accordion Fold / 手风琴折叠 | 循环 | 4.8s | 把长纸带逐段折成一个小纸包 |
| `workspace` | Pop-up Studio / 弹出工作间 | 单次 | 3.4s | 从平纸包展开两面墙和一张小桌 |
| `success` | Self High-five / 自我击掌 | 单次 | 2.8s | 蹲下、跳起、两只爪在头顶击掌 |
| `error` | Yarn Tangle / 毛线缠住 | 循环 | 4.4s | 拉扯毛线、被缠住、停顿后再试 |

## 4. 逐状态设计

### `idle` — Calm Calibration

- **语义**：主人在电脑前，当前没有运行任务。
- **配件**：无。待机不能让角色看起来正在工作。
- **身体动作**：身体每个循环只上浮/回落 `1px`；双爪保持松弛；腿和身体同步，不能产生漂浮感。
- **眼睛动作**：保留鼠标追踪能力；无鼠标目标时回到固定坐标；循环后段短眨眼一次。
- **节奏**：5.6 秒 `step-end` 循环，至少包含呼吸、停顿和眨眼三段可见变化。
- **验收读法**：缩到约 96px 时应读作“清醒但不打扰”，不能读成思考、挥手或工作。
- **现有实现**：[`src/animations/calm-calibration.svg`](../src/animations/calm-calibration.svg)。

### `away` — Blanket Drag

- **语义**：主人短暂离开，Maclawd 正在进入休眠链。
- **配件**：紫色软毛毯，黄色小补丁与流苏帮助小尺寸辨认。
- **身体动作**：先伸懒腰，再回身抓住毛毯一角，把毛毯拖到身体旁并逐渐眯眼。
- **节奏**：3.6 秒单次动作，运行时播放一轮；实验室为方便评审自动重播。
- **验收读法**：必须明显比 `idle` 更疲惫，但不能提前完全躺倒。
- **现有实现**：[`src/animations/blanket-drag.svg`](../src/animations/blanket-drag.svg)。

### `sleeping` — Blanket Burrito

- **语义**：主人长时间没有操作。
- **配件**：沿用 `away` 的同一条毛毯，不增加底座、灯或睡眠符号。
- **身体动作**：身体压低并被毛毯包住；毛毯上沿随呼吸抬起一格，流苏稍晚跟随。
- **眼睛动作**：双眼保持闭合变换，不增加 `Z` 字或睡眠气泡。
- **节奏**：5.6 秒循环；身体与毛毯使用轻微错相呼吸。
- **验收读法**：必须读成“舒服地裹着睡眠”，而不是受伤、错误或死亡。
- **现有实现**：[`src/animations/blanket-burrito.svg`](../src/animations/blanket-burrito.svg)。

### `waking` — Blanket Pop

- **语义**：主人重新移动鼠标或操作电脑。
- **配件**：沿用 `sleeping` 的毛毯，并拆成左右两块可掀开的折面。
- **身体动作**：身体快速弹起 `2px`，双爪向外撑开，两块毛毯像帘子一样落到两侧，最后回到中性站姿。
- **眼睛动作**：睁眼后有一次极短的确认眨眼。
- **节奏**：2.2 秒单次动作，结尾与 `idle` 的第一帧一致。
- **验收读法**：轻快但不过度惊吓，不使用飞散零件或冲击线。
- **现有实现**：[`src/animations/blanket-pop.svg`](../src/animations/blanket-pop.svg)。

### `thinking` — Shell Shuffle

- **语义**：收到任务，尚未进入明确工具调用。
- **道具**：青、黄、紫三枚像素贝壳和一条浅色小布。
- **身体动作**：Maclawd 左右偏移查看三枚贝壳，双爪指向当前对象，最后一起举起中间贝壳。
- **眼睛动作**：视线按左—中—右移动，选定后回中并眨眼。
- **节奏**：5.6 秒循环，左中右依次跳起，然后举起选项并复位。
- **验收读法**：读成“认真比较后做决定”，不像机械转档或界面选项。
- **现有实现**：[`src/animations/shell-shuffle.svg`](../src/animations/shell-shuffle.svg)。旧的 Inference Dial 保留为历史原型。

### `working` — Token Knitting

- **语义**：Agent 正在使用工具，但活动类型未知或没有专属修饰动作。
- **道具**：一团青色毛线、两根粗像素织针和一条会逐步变长的条纹彩带。
- **身体动作**：双爪与织针交替上下，身体跟随四拍起伏；彩带变长后向右侧举起检查。
- **眼睛动作**：编织时向下看，检查时向右看，收尾眨眼。
- **节奏**：4.8 秒循环，四次编织拍后进入展示与复位。
- **现有实现**：[`src/animations/token-knitting.svg`](../src/animations/token-knitting.svg)。旧的 Reasoning Gearbox 保留为历史原型。
- **验收读法**：必须是持续处理，而不是思考、测试结果或庆祝。

### `delegating` — Hatchling Parade

- **语义**：一个或多个子 Agent 正在执行任务。
- **道具/角色**：三只体型更小、颜色不同的像素小蟹助手。
- **身体动作**：Maclawd 低头逐个唤醒小蟹，再用右爪指向任务方向，视线跟随它们出发。
- **节奏**：4.8 秒循环；三只小蟹每隔一拍出现，错相向右移动并离场。
- **现有实现**：[`src/animations/hatchling-parade.svg`](../src/animations/hatchling-parade.svg)。
- **验收读法**：必须读成“分配并行工作”，不使用耳机、球、指挥棒或抛接动作。

### `needs_owner` — Stuck Jar

- **语义**：Agent 等待权限、回答或关键确认。
- **道具**：一只小玻璃罐，深色罐盖与粉色内容物保证小尺寸识别。
- **身体动作**：双爪抱住罐子，身体左右用力两次，第二次后压低停顿，再把罐子向画面前方递出。
- **眼睛动作**：用力时压窄，递出时恢复并直视主人。
- **节奏**：4.2 秒循环，拧、反向拧、再拧、放弃、递出并复位。
- **现有实现**：[`src/animations/stuck-jar.svg`](../src/animations/stuck-jar.svg)。
- **验收读法**：必须读成“需要主人操作”，不能像错误或普通通知。

### `compacting` — Accordion Fold

- **语义**：Agent 正在压缩上下文或整理会话。
- **配件**：一条相互连接的四色四段纸带，没有机器、资料匣或 UI 面板。
- **身体动作**：左右爪交替把纸带向中间折入，最后压成一个小纸包并低头检查。
- **节奏**：4.8 秒循环；折叠、检查、展开复位都在同一条连续纸带上完成。
- **验收读法**：必须读成“把长内容折叠得更紧凑”，不使用扫帚、搬运或压缩机动作。
- **现有实现**：[`src/animations/accordion-fold.svg`](../src/animations/accordion-fold.svg)。

### `workspace` — Pop-up Studio

- **语义**：创建新分支、Worktree 或独立工作空间。
- **配件**：一个扁平纸艺包，向左右展开成青色墙、黄色墙和一张深色小桌。
- **身体动作**：双爪拉开纸墙，小桌从中间竖起；角色分别敲两侧确认稳固。
- **节奏**：3.4 秒单次动作；完成后自然转入 `working`。
- **验收读法**：必须读成“快速搭好一个新工作间”，不使用纸箱、搬运或固定机器动作。
- **现有实现**：[`src/animations/pop-up-studio.svg`](../src/animations/pop-up-studio.svg)。

### `success` — Self High-five

- **语义**：任务已可靠完成。
- **配件**：无。成功首先是角色自己的情绪表演。
- **身体动作**：先蹲下蓄力，再向上轻跳 `2px`，两只爪在头顶相碰，落地后收回。
- **眼睛动作**：落地后一次快速眨眼。
- **节奏**：2.8 秒单次动作；运行时播放一轮后进入 `idle`。
- **验收读法**：必须读成“真的很高兴完成了”，不使用印章、烟花、星星或漂浮符号。
- **现有实现**：[`src/animations/self-high-five.svg`](../src/animations/self-high-five.svg)。

### `error` — Yarn Tangle

- **语义**：工具或命令执行失败，等待下一步恢复。
- **配件**：一团红色毛线与一条始终连续的曲折毛线圈，避免成为红色 UI 边框。
- **身体动作**：向左拉线、被线圈缠住、向右反拉、压低停顿，再松开复位。
- **眼睛动作**：眼睛保持原矩形，只通过位移和压缩表达“事情不对劲”。
- **节奏**：4.4 秒循环；错误持续时重复尝试，收到恢复事件后退出。
- **验收读法**：必须读成“被具体问题缠住但还在尝试”，不使用烟、漂浮感叹号或红色大叉。
- **现有实现**：[`src/animations/yarn-tangle.svg`](../src/animations/yarn-tangle.svg)。

## 5. 转场合同

- 每个循环动画的第一帧与最后一帧必须一致。
- `away → sleeping → waking → idle` 必须共享同一条毛毯的颜色、补丁和折面语言。
- `thinking → working` 先放下贝壳，再让毛线团与织针入场。
- `working → needs_owner` 先停止编织并收起彩带，再把罐子递入双爪。
- `workspace`、`success` 是单次动作，结尾分别进入 `working`、`idle`。
- `error`、`needs_owner` 由外部事件解除，不自动宣告成功。
- 状态切换可使用最长 `120ms` 的透明度交叉淡入，但不能依赖模糊或运动拖影。

## 6. 动效与可访问性合同

- 使用 CSS keyframes；默认采用 `step-end` 或离散阶梯，不使用连续模糊插值。
- 身体组常规位移不超过 `1px`，仅 `waking` 和 `success` 可达到 `2px`。
- 除眼睛闭合变换外，身体和眼睛源矩形不得改变。
- 正常模式下每个循环至少包含一次组件级变化，不能是近乎静态的多帧复制。
- `prefers-reduced-motion` 下停止道具、小蟹和身体循环，只保留一次低频眨眼。
- 所有状态需在约 96px 显示尺寸下进行语义审查。

## 7. 实现顺序

1. `idle`、`thinking`：确定角色无配件和轻配件时的动作基线。
2. `working`：沿用并校正现有推理齿轮箱。
3. `needs_owner`、`success`、`error`：已完成状态反馈闭环。
4. `away`、`sleeping`、`waking`：已完成连续毛毯休眠链。
5. `delegating`、`compacting`、`workspace`：已完成系统和并行状态。

工作活动修饰与互动动作已单独记录在
[`activity-modifiers.md`](activity-modifiers.md) 和
[`interaction-actions.md`](interaction-actions.md)，不会混入主状态优先级。
